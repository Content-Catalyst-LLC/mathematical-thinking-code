# Schema Notes

The SQL schema models mathematical patterns as records with domain, type, structure, evidence, proof status, and interpretive warning.

## Core tables

- `pattern_domain`: broad mathematical domains of pattern study.
- `pattern_record`: individual pattern records with evidence and proof status.
- `invariant_record`: invariant metadata and completeness warnings.
- `counterexample_record`: claims, objects, and lessons for false-pattern discipline.

## Professional use cases

- mathematical education;
- theorem and conjecture tracking;
- proof-assistant formalization planning;
- reproducible computational exploration;
- pattern and invariant cataloging;
- counterexample documentation.
