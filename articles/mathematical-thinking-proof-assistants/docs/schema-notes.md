# Schema Notes

The SQL schema models proof-assistant systems, formalization projects, theorem-statement audits, trust boundaries, proof-assistant skills, proof layers, AI formalization workflows, and responsible verification checks.

## Core tables

- `proof_assistant_system`: system, foundation, strength, use, trust note.
- `formalization_project`: proof assistant, foundation, domain, purpose.
- `theorem_statement_audit`: informal statement, formal summary, hypotheses, intended meaning review, risk.
- `proof_trust_boundary`: trusted component and review question.
- `proof_assistant_skill`: skill and review question.
- `proof_layer`: human role, machine role, and risk by layer.
- `ai_formalization_workflow`: AI role, proof assistant role, human role, risk, mitigation.
- `responsible_verification_check`: define-state-prove-check-interpret review.

## Professional use cases

- proof-assistant literacy;
- theorem-statement audits;
- formalization project planning;
- trust-boundary documentation;
- AI-to-proof-assistant workflow review;
- responsible verification governance.
