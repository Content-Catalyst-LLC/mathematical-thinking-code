# Schema Notes

The SQL schema models category records, functor records, natural transformation records, universal properties, diagrams, abstraction risks, and responsible abstraction checks.

## Core tables

- `category_record`: objects, morphisms, preserved structure, composition, risk.
- `functor_record`: source/target categories, mappings, preservation, forgotten or added structure.
- `natural_transformation_record`: component description, naturality condition, coherence review.
- `universal_property_record`: diagram shape, universal role, uniqueness condition, examples.
- `diagram_record`: paths, commutativity condition, interpretation.
- `abstraction_risk`: risk, problem, mitigation.
- `responsible_abstraction_check`: objects-arrows-structure-universality-responsibility workflow.

## Professional use cases

- category-level abstraction audits;
- functorial preservation review;
- universal-property teaching;
- diagrammatic reasoning checks;
- applied category theory modeling review;
- responsible abstraction governance.
