# Validation Plan

## Tally and correspondence

- Confirm one mark corresponds to one object.
- Preserve collection identifiers.
- Avoid treating count as full context.

## Sequence and pattern

- Record finite values.
- Compute finite differences where relevant.
- Label evidence status separately from proof status.
- Include deliberately misleading finite patterns.

## Cycles

- Record period, starting point, and modular rule.
- Distinguish observed rhythm from imposed coding scheme.

## Representation

- State what each representation preserves.
- State what each representation omits.
- Require interpretation notes for patterns used in applied settings.

## Responsible-use note

Synthetic examples in this repository are for methods demonstration. Real historical, cultural, educational, or data-centered work requires careful sourcing, consent where relevant, and contextual interpretation.
