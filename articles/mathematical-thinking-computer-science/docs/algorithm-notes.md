# Algorithm Notes

Algorithms should be documented as mathematical objects.

## Recommended fields

- input domain;
- output specification;
- preconditions;
- postconditions;
- invariants;
- termination argument;
- complexity bound;
- correctness proof method;
- responsible-use note.

## Audit questions

- What does the algorithm guarantee?
- Under what assumptions?
- How is correctness proven?
- How does cost grow?
- What should not be inferred from the output?
