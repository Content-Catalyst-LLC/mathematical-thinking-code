# Schema Notes

The SQL schema models computational concepts, algorithm specifications, complexity cases, proof obligations, automata cases, type examples, probability cases, linear algebra examples, modular arithmetic examples, and representation warnings.

## Core tables

- `computational_concept`: major mathematical structures used in computer science.
- `algorithm_specification`: input/output, invariant, complexity, correctness, and responsible-use metadata.
- `proof_obligation`: proof claims and methods tied to algorithms.
- `complexity_case`: asymptotic growth examples.
- `automata_case`: finite-machine definitions and recognized languages.
- `type_example`: type structures and mathematical analogies.
- `representation_warning`: responsible computing warnings and mitigations.

## Professional use cases

- computer-science foundations instruction;
- algorithm-review documentation;
- proof-obligation tracking;
- responsible AI/software review;
- data-structure and type-design audits;
- complexity/scalability demonstrations.
