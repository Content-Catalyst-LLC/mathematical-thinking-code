# Validation Plan

## Algorithms

- Confirm input domain and output specification.
- Record invariant and proof method.
- Separate test evidence from proof evidence.
- Check boundary cases.
- Confirm complexity claim.

## Data structures and types

- Document invariants.
- Record valid and invalid operations.
- Identify recursive structure where present.
- Distinguish representation from meaning.

## Graphs and automata

- Validate node/edge or state/transition definitions.
- Check reachability and accepted-language examples.
- Track direction, weights, and provenance where relevant.

## Probability and linear algebra

- Define sample space and assumptions.
- Record uncertainty and calibration concerns.
- Document vector/matrix representation and interpretation limits.

## Responsible-use note

Synthetic examples in this repository are for methods demonstration. Applied computer-science systems require proof, testing, verification, security review, domain validation, affected-community review, and consequences analysis.
