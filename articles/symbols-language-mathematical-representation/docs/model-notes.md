# Model Notes

This article treats mathematical representation as a structured relation among object, symbol, preserved structure, and interpretation.

The companion repository models representation through five linked ideas:

1. **Mathematical object** — the number, function, graph, theorem, definition, or system being represented.
2. **Representation** — the notation, diagram, formula, table, matrix, code, or formal language used to express it.
3. **Preserved structure** — the relation, operation, quantity, dependency, or invariant retained by the representation.
4. **Omitted detail** — what the representation leaves out.
5. **Translation** — the rule by which one representation is transformed into another.

The repository treats representation as an object of audit: a mathematical representation should be examined for what it makes visible and what it hides.
