# Notation Notes

Notation is mathematical infrastructure. It shapes what can be expressed, manipulated, generalized, formalized, and computed.

## Notation audit questions

- What object does the notation represent?
- What domain is assumed?
- Is the variable free, bound, arbitrary, unknown, or a parameter?
- Does equality mean exact equality, equivalence, congruence, isomorphism, or approximation?
- Does the notation hide assumptions?
- Does the notation transfer safely to computation?
- What would a formal proof assistant need to know that prose currently leaves implicit?

## Haskell relevance

Haskell is included prominently because algebraic data types make symbolic structure explicit. Expressions, propositions, proof trees, abstract syntax trees, and graph representations can be modeled directly as typed data.
