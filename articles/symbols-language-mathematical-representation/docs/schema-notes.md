# Schema Notes

The SQL schema models mathematical representation.

## Core tables

- `mathematical_object`: objects such as numbers, functions, graphs, theorems, definitions, and systems.
- `representation`: symbolic, tabular, diagrammatic, formal, or computational forms.
- `translation`: relations between representations.
- `notation_symbol`: common mathematical symbols and ambiguity warnings.
- `representation_warning`: known risks and mitigations.

## Professional use cases

- mathematical knowledge architecture;
- notation and representation audits;
- theorem and proof metadata;
- formalization planning;
- graph and matrix representation tracking;
- model transparency documentation;
- educational representation comparison.
