# Schema Notes

The SQL schema models sets, relations, functions, warnings, and model structures.

## Core tables

- `mathematical_set`: named finite sets with definition rules and domain notes.
- `relation_record`: relation pairs with relation type and interpretation.
- `function_record`: function mappings with domain, codomain, rule, and validation note.
- `mapping_warning`: warnings for boundaries, codomains, categories, and non-function relations.
- `model_structure`: toy examples combining object sets, relations, functions, and assumptions.

## Professional use cases

- discrete mathematics demonstrations;
- database and schema pedagogy;
- relation-property audits;
- function validation;
- domain/codomain governance;
- AI and data-system mapping review.
