# Schema Notes

The SQL schema models foundation views, mathematical structures, formal systems, transformation maps, model interpretations, proof-assistant layers, and abstraction warnings.

## Core tables

- `foundation_view`: foundational/philosophical view, central question, strength, limitation.
- `mathematical_structure`: objects, operations, laws, and preservation map.
- `formal_system`: language, axioms, inference rules, intended use, limitations.
- `transformation_map`: structural map and what it preserves.
- `model_interpretation`: formal structure, interpretations, assumption risks.
- `proof_assistant_layer`: human and machine roles in formalization.
- `abstraction_warning`: responsible abstraction risks and mitigations.

## Professional use cases

- foundations education;
- structural mathematics teaching;
- formal verification literacy;
- model interpretation audits;
- responsible abstraction review;
- mathematical knowledge-architecture design.
