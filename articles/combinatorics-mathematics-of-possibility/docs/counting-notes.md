# Counting Notes

A reliable combinatorial count should explain why every valid object is counted, why no invalid object is counted, and why no valid object is counted more than intended.

## Common counting risks

- using permutations when combinations are needed;
- ignoring repetition rules;
- double-counting overlapping cases;
- missing edge cases;
- treating non-equally-likely outcomes as equally likely;
- failing to account for symmetry;
- applying a formula without checking assumptions.

## Recommended audit habit

Before computing, write a one-sentence definition of the possibility space.
