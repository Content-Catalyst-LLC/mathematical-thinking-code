# Model Notes

This article treats abstraction as structure selection and generalization as the controlled extension of claims across a class of objects.

The companion repository models five linked ideas:

1. **Abstraction** — preserve selected structure while omitting irrelevant detail.
2. **Equivalence** — group different presentations that represent the same mathematical object for a given purpose.
3. **Mapping** — move objects between contexts while tracking whether relevant structure is preserved.
4. **Invariant** — identify properties that remain stable under allowed transformations.
5. **Counterexample** — refine or refute overgeneralized claims.

The workflow separates computational evidence from proof. Finite case checks, generated tables, and invariant calculations can support conjecture and teaching, but they do not automatically establish universal claims.
