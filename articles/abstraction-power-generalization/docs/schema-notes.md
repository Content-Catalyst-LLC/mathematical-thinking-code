# Schema Notes

The SQL schema separates abstraction, generalization, mapping, invariant, and counterexample records.

## Core entities

- `abstraction`: what is preserved and what is omitted.
- `generalization`: a claim extended across a domain under assumptions.
- `mapping`: a transformation between structures and the relation it preserves.
- `invariant`: a property stable under a transformation class.
- `counterexample`: an object showing where a generalization fails.

## Professional use cases

- theorem and assumption tracking;
- teaching abstraction and proof discipline;
- formalization planning;
- mathematical knowledge architecture;
- computational audit of examples and counterexamples.
