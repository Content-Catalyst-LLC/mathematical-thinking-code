# Discovery Notes

This article treats conjecture as disciplined mathematical imagination.

A useful discovery workflow is:

1. observe examples;
2. formulate a conjecture;
3. test special and boundary cases;
4. search for counterexamples;
5. identify structure;
6. attempt proof;
7. revise if the proof fails or a counterexample appears;
8. generalize when the proof reveals deeper structure.

The repository tracks this workflow through conjecture metadata, evidence records, counterexamples, proof attempts, graph invariants, sequence audits, and proof-status summaries.
