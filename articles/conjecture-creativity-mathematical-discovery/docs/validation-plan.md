# Validation Plan

## Conjecture metadata

- Every conjecture must have a domain.
- Every conjecture must have a proof status.
- Evidence must be linked to a conjecture.
- Counterexamples must identify the failure mode.
- Proof attempts must document method and lesson.

## Finite evidence

- Treat finite checks as evidence, not proof.
- Record tested range or scope.
- Distinguish exhaustive finite verification from sampled testing.
- Include deliberately false or misleading examples.

## Graph invariant discovery

- Record graph assumptions: finite, undirected, simple, weighted, directed, etc.
- Treat degree sequence as incomplete unless full classification is proved.
- Document search range and graph-generation method.

## Computational discovery

- Fix random seeds where randomness is used.
- Record parameter ranges.
- Check code assumptions against mathematical assumptions.
- Label AI-generated or computationally suggested claims as conjectural until verified.
