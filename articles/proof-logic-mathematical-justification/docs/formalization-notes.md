# Formalization Notes

This article folder is not itself a Lean, Rocq/Coq, Isabelle, or Agda formalization. It is a planning scaffold for formalization.

## Formalization preparation checklist

- State the theorem precisely.
- Identify the domain and all assumptions.
- Determine whether definitions already exist in the target library.
- Break the proof into lemmas.
- Record dependencies explicitly.
- Identify whether the proof is direct, inductive, constructive, by contradiction, or by counterexample.
- Translate informal proof steps into formal proof obligations.
- Check whether the theorem statement matches the intended informal meaning.

## Candidate formalization examples

- Even-square lemma.
- Sum of first \(n\) positive integers.
- Counterexample to bounded-sequence convergence.
- Graph degree sequence as an incomplete invariant.
- Modus ponens and induction-rule metadata.
