# Invariant Schema

An invariant is a property that remains stable under a specified class of transformations.

## Graph examples

- vertex count
- edge count
- degree sequence
- connected component count
- cycle rank
- chromatic number
- spectrum
- automorphism group

## Sequence examples

- parity pattern
- residue period
- growth class
- finite-difference behavior
- recurrence order

## Schema pattern

```text
object -> transformation class -> invariant -> evidence/proof note
```

A computed invariant can support conjecture, classification, and comparison. It does not automatically prove isomorphism or equivalence unless the invariant is known to be complete for the relevant object class.
