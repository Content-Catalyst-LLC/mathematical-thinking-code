# Validation Plan

## Sequence analysis

- Verify sequence prefixes against definitions.
- Distinguish observed periodicity from proven periodicity.
- Compare finite differences and residues across implementations.

## Graph invariants

- Confirm edge-list parsing.
- Validate vertex and edge counts manually on small graphs.
- Check degree-sequence calculations across Python, Rust, Go, and C++.
- Treat degree sequence as a partial invariant, not a complete isomorphism test.

## SQL metadata

- Keep patterns, mathematical objects, invariants, analogies, and counterexamples logically distinct.
- Use schema constraints to preserve basic integrity.
- Use query outputs to audit interpretive claims.

## Interpretation

The scaffold supports mathematical exploration and reproducibility. It does not replace proof, complete classification, formal verification, or expert mathematical review.
