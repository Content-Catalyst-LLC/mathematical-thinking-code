# Proof Notes

Proof connects a procedure to a specification.

## Common methods

- loop invariant;
- induction;
- structural induction;
- greedy invariant;
- contradiction;
- case analysis;
- data-structure invariant.

## Correctness distinction

- Partial correctness: if the algorithm terminates, the result satisfies the specification.
- Termination: the algorithm stops.
- Total correctness: partial correctness plus termination.
