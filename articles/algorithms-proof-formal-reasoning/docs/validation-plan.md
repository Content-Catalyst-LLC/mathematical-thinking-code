# Validation Plan

## Specifications

- Confirm input domain.
- Confirm precondition and postcondition.
- Record failure behavior.
- Record responsible-use note.

## Proof obligations

- Identify proof method.
- State invariant, induction hypothesis, or greedy invariant.
- Separate testing evidence from proof evidence.
- Document assumptions.

## Termination

- Identify a decreasing measure.
- Identify a lower bound.
- Check edge cases.

## Complexity

- Identify input size.
- Identify dominant operation.
- Document asymptotic claim and model.

## Responsible-use note

Synthetic examples in this repository are for methods demonstration. Applied use requires complete proofs, tests, verification, security review, code review, domain validation, stakeholder review, and consequences analysis.
