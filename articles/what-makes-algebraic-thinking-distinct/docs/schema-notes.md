# Schema Notes

The SQL schema models algebraic-thinking metadata.

## Core tables

- `algebraic_object`: expressions, equations, functions, identities, sequences, and structures.
- `variable_role`: unknowns, generalized numbers, inputs, outputs, parameters, indices, and coefficients.
- `transformation_rule`: source/target forms, preserved meaning, and risk notes.
- `equation_audit`: equation type, interpretation, and verification method.
- `function_representation`: verbal, table, formula, graph, and code representations.
- `algebraic_misconception`: diagnostic signals and interventions.

## Professional use cases

- algebra education research;
- symbolic computation design;
- expression-tree analysis;
- transformation validity auditing;
- equation verification workflows;
- function representation comparison;
- AI-generated algebra review.
