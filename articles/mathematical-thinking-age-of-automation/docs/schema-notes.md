# Schema Notes

The SQL schema models automation tasks, tools, verification records, automation risks, human judgment skills, and proof-assistant layers.

## Core tables

- `automation_task`: task, tool, object, output type, assumptions, verification method.
- `automation_tool`: tool category, strength, human review, failure mode.
- `verification_record`: evidence standard, trust boundary, interpretation note.
- `automation_risk`: mathematical problem and mitigation.
- `human_judgment_skill`: skill, context, review question.
- `proof_assistant_layer`: human and machine roles across formalization.

## Professional use cases

- mathematical automation audits;
- classroom tool-literacy design;
- AI-output review workflows;
- symbolic and numerical verification checklists;
- proof-assistant trust-boundary review;
- responsible mathematical modeling governance.
