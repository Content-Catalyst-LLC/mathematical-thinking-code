# Haskell Companion Scaffold

This Haskell folder supports **Mathematical Thinking in an Age of Automation**.

Haskell is useful here because automation types, output types, evidence standards, risk types, and human responsibilities can be modeled explicitly with algebraic data types.

## What this folder demonstrates

- automation systems as typed alternatives;
- output types as typed categories;
- evidence standards as typed review requirements;
- automation risks as typed records;
- human responsibility as an explicit field;
- specify-compute-verify-interpret as a reusable audit sequence.

## Run

```bash
cd haskell
make run
```

This writes generated artifacts to:

```text
../outputs/tables/haskell_automation_records.csv
../outputs/tables/haskell_automation_risks.csv
../outputs/tables/haskell_specify_compute_verify_interpret.csv
```

## Professional note

These examples support mathematical automation audits, proof-assistant literacy, AI-output review, symbolic/numerical verification, and responsible tool use. They do not replace mathematical proof, formal verification, model validation, numerical analysis, or ethical review.
