# Schema Notes

The SQL schema models historical practices, historiographic risks, mathematical transmissions, notation history, proof styles, and canon risks.

## Core tables

- `historical_practice`: practice, object, medium, method, meaning, caution.
- `historiographic_risk`: problem and mitigation.
- `mathematical_transmission`: preservation and transformation across contexts.
- `notation_history`: notation/medium, function, effect, anachronism warning.
- `proof_style`: proof style, authority basis, and caution.
- `canon_risk`: canon problem and responsible response.

## Professional use cases

- mathematical historiography;
- notation-history education;
- proof-style comparison;
- global mathematics curriculum design;
- canon-risk audits;
- responsible historical interpretation workflows.
