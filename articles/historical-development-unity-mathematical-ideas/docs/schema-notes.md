# Schema Notes

The SQL schema models mathematical ideas, historical layers, cross-field connections, transformation/invariant pairs, proof/algorithm/model distinctions, and responsible generalization warnings.

## Core tables

- `historical_layer`: broad historical layer and dominant unifying idea.
- `mathematical_idea`: idea, field, representation, transformation, invariant, interpretation.
- `cross_field_connection`: source idea, target idea, connection type, preserved structure, and caution.
- `transformation_invariant`: field, transformation, invariant, meaning, and example.
- `proof_algorithm_model_connection`: evidence standards and risks.
- `responsible_generalization_warning`: warnings and mitigations.

## Professional use cases

- mathematical-history teaching;
- idea-network mapping;
- cross-field mathematical concept design;
- model transfer review;
- proof/algorithm/model literacy;
- responsible generalization audits.
