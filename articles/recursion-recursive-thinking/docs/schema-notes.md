# Schema Notes

The SQL schema models recursive definitions, recurrence records, grammar rules, recursive algorithms, recursive model audits, and recursion warnings.

## Core tables

- `recursive_definition`: base case, recursive rule, termination measure, and interpretation.
- `recurrence_record`: recurrence metadata and proof status.
- `grammar_rule`: base and recursive grammar productions.
- `recursive_algorithm`: base cases, reductions, combination rules, complexity, and correctness notes.
- `recursive_model_audit`: update rules, stopping conditions, and risk notes.
- `recursion_warning`: warning and mitigation records.

## Professional use cases

- recursion instruction;
- proof and induction pedagogy;
- recursive data-structure documentation;
- algorithm correctness review;
- recursive model risk audit;
- AI and feedback-system governance.
