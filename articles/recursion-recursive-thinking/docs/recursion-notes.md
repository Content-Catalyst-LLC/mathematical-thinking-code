# Recursion Notes

Recursion defines or solves a structure in terms of smaller instances of itself.

## Core requirements

- base case;
- recursive rule;
- decreasing measure or structural reduction;
- stopping condition;
- combination rule;
- invariant or correctness argument.

## Key risks

- missing base case;
- wrong base case;
- recursive step does not reduce the problem;
- repeated subproblems;
- cycles in graph recursion;
- error propagation in recursive models;
- outputs feeding back into future inputs without validation.
