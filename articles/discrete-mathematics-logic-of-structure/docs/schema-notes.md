# Schema Notes

The SQL schema models discrete objects, relation edges, algorithms, proof patterns, and structure warnings.

## Core tables

- `discrete_object`: vertices, propositions, states, and other discrete units.
- `relation_edge`: graph, transition, and proof-dependency edges.
- `algorithm_audit`: algorithm input/output structures, invariants, and complexity notes.
- `proof_pattern`: proof forms, required structures, and common risks.
- `structure_warning`: warnings and mitigations for discrete structure interpretation.

## Professional use cases

- discrete mathematics demonstrations;
- graph and network pedagogy;
- algorithm documentation;
- proof-pattern education;
- data-system schema review;
- AI and decision-system structure audits.
