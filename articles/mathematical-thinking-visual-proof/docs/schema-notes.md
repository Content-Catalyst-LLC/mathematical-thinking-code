# Schema Notes

The SQL schema models visual proof records, diagram relations, visual risks, accessibility reviews, visual workflows, and proofs without words.

## Core tables

- `visual_proof_record`: title, domain, visual role, visible structure, proof status, generalization question.
- `diagram_relation`: visual feature, mathematical relation, proof requirement.
- `visual_risk`: risk, problem, mitigation.
- `accessibility_review`: visual dependency, alternative description, accessibility note.
- `visual_workflow`: see-abstract-prove-interpret stages.
- `proof_without_words_example`: visual device, mathematical idea, missing-words prompt.

## Professional use cases

- visual proof audits;
- diagrammatic reasoning review;
- dynamic geometry observation records;
- proof-without-words teaching;
- graph drawing caution review;
- accessibility and responsible visualization governance.
